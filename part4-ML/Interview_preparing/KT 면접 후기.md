​	주제 : blockchain 기술을 이용한 암호화폐, 법정화폐로 사용할 수 있는가?

​	최대한 그냥 나가서 발표하려고 했으나, 헛소리는 지향하려고 했음. 싸움을 하지는 않았음

####   자소서 기반 직무면접

​		자소서 기반 직무면접은 정말 자기소개도 안시키고 그냥 자소서에 대해서 바로 들어감

​		PT면접(문제를 어떻게 풀었느냐를 설명)이 끝나면 바로 자연스럽게 자소서에 대해 묻기 때문에 자기소개 할 필요와 시간이 없음

​		실제적으로 AI에 대한 내용을 자기소개에 많이 적었더니 프로젝트를 많이 했다고 칭찬들음

​		내가 프로젝트에서 어떤 부분을 했는지 물어봄 : 프로젝트가 크게 웹(UI & Deploy) 파트와 내부 알고리즘 구현 파트로 나눠져 있었는데, 나는 내부 알고리즘 쪽을 맡았다고 이야기 함

​		이야기 하며 너무 장황하게 하지 않으려고 했으며, 묻는 질문에 대한 정확한 답을 하려고 최대한 노력함

​		내부 알고리즘을 수행하면서 cloud platform (GCP)를 내가 직접 시스템을 scratch부터 구현하여 framework들을 설치했다는 것을 더 어필함 : cloud에 대한 관심이 많은 것 같음 : 그래서 확실히 내가 했다고 하고, 특히 GPU 환경을 활용한 시스템 엔지니어링을 배웠으며, linux 환경에 익숙해졌다고 어필함

​		아무래도 통신사인데다가 대기업이다보니 주 언어가 Python인데 문제가 있지 않겠냐라고 이야기함(대부분 enterprise 환경에서 작업을 하려면 JAVA를 쓴다라는 뉘앙스)

​		=> 이전에 자바 Naive MVC 패턴으로 웹 프로그래밍을 해 보았으며, 현재 python을 이용한 Django 웹 프레임워크를 공부하면서 객체지향 언어에 대해서 적응력을 높여가고 있음을 어필함

​		또한, 언어는 그냥 도구일 뿐이고, 하나에 대한 깊이가 깊어지면 다른 언어들은 배우기 쉽고, 현재 그것을 느끼고 있음을 어필함 : 이러한 새로운 도전을 즐기고 공부하는 것을 즐김을 어필함

​		그 외에는 자소서 기반으로 물어봤던 것 같음 : 시간이 엄청 길지는 않아서 완전 세세하게 자소서를 다 물어보지는 않음
